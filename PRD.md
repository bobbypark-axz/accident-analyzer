# 교통사고 과실비율 AI 분석 서비스 — PRD

## 1. 제품 개요

### 1.1 제품명
교통사고 과실비율 AI 분석기

### 1.2 한 줄 요약
블랙박스 영상 또는 텍스트 설명만으로 교통사고 과실비율을 AI가 분석하고, 사고 장면을 2D로 재현해주는 웹 서비스

### 1.3 배경 및 문제 정의
- 교통사고 발생 시 과실비율 판단은 보험사 합의 또는 법원 판결까지 오랜 시간 소요
- 일반 운전자는 자신의 과실비율을 사전에 예측하기 어려움
- 블랙박스 영상이 있어도 전문 지식 없이는 사고 상황을 객관적으로 판단하기 힘듦
- 기존 서비스는 텍스트 기반 분석만 제공하거나, 영상 분석 기능이 없음

### 1.4 목표 사용자
| 사용자 유형 | 니즈 |
|------------|------|
| 사고 당사자 | 내 과실이 얼마나 되는지 빠르게 파악하고 싶음 |
| 보험 가입자 | 보험사 합의 전 예상 과실비율을 알고 협상력을 높이고 싶음 |
| 운전 학습자 | 다양한 사고 유형의 과실 기준을 학습하고 싶음 |

### 1.5 핵심 가치 제안
1. **즉시성** — 텍스트 입력 또는 영상 업로드 후 1분 이내 분석 결과 제공
2. **객관성** — 도로교통법 조항과 실제 판례 기반 분석
3. **시각성** — 2D 사고 재현 애니메이션으로 사고 상황 직관적 이해
4. **접근성** — 법률 전문 지식 없이도 누구나 이해 가능한 결과 제공

## 2. 핵심 기능

### 2.1 텍스트 기반 사고 분석 (기본 모드)
| 항목 | 상세 |
|------|------|
| 입력 | 사고 상황 텍스트 설명 (자유 형식 또는 템플릿 선택) |
| 처리 | Claude AI가 도로교통법 + 판례 기반 분석 |
| 출력 | 과실비율, 사고 요약, 적용 법규, 유사 판례, 참고사항 |
| 부가 | 2D 사고 재현 애니메이션 (AI가 사고 구조 데이터 생성 → SVG 렌더링) |

**사고 유형 템플릿 (빠른 입력):**
- 교차로 사고 (직진 vs 좌회전)
- 추돌 사고 (전방 급정거)
- 차선 변경 사고
- 주차장 사고
- 보행자 사고

### 2.2 영상 기반 정밀 분석 (상세 모드)
| 항목 | 상세 |
|------|------|
| 입력 | 블랙박스 영상(MP4, AVI, MOV) 또는 사고 이미지(JPG, PNG) + 선택적 텍스트 설명 |
| 처리 | YOLOv8s 객체 감지 → ByteTrack 추적 → 충돌 감지 → Claude AI 종합 분석 |
| 출력 | 사고 순간 프레임 + YOLO 감지 결과 + 과실비율 분석 + 2D 재현도 |

**영상 분석 파이프라인:**
```
영상 입력 → 1초 간격 프레임 샘플링
         → YOLOv8s 객체 감지 (차량, 보행자, 신호등)
         → ByteTrack 차량 추적 (ID 유지)
         → 충돌 감지 (IoU + 속도 급변 조합)
         → 사고 순간 프레임 자동 탐지 (연속 2프레임 충돌 확인)
         → Claude AI 종합 분석 (이미지 + YOLO 데이터 + 텍스트 설명)
```

**감지 항목:**
- 차량 종류 (승용차, 트럭, 버스, 오토바이)
- 보행자
- 신호등 색상 (빨강, 노랑, 초록)
- 차선 (Canny + HoughLines)
- 충돌 여부 및 지점
- 전복 의심 (바운딩 박스 비율 분석)
- 차선 이탈

### 2.3 2D 사고 재현 애니메이션
| 항목 | 상세 |
|------|------|
| 렌더링 | 인라인 SVG + SMIL 애니메이션 |
| 도로 유형 | 교차로(십자), 직선도로, 커브 |
| 차량 표현 | 차종별 상세 SVG (승용차: 유리창+바퀴+라이트, 오토바이: 핸들+바퀴, 보행자: 사람 형태) |
| 좌표 시스템 | 슬롯 기반 (AI가 N1, S2, C 등 미리 정의된 위치를 선택) |
| 애니메이션 | 차량이 시작 위치에서 충돌 지점으로 이동 → 충돌 임팩트 이펙트 |

**슬롯 맵:**
```
교차로:                    직선도로:
     N1                    L1 ── L2 ── L3 (상행)
     N2                    R1 ── R2 ── R3 (하행)
     N3                    F1 ── F2 ── F3 (추돌용)
W1 W2 W3 [C] E3 E2 E1
     S3
     S2
     S1
```

### 2.4 분석 결과 표시
| 구성 요소 | 표시 방식 |
|-----------|----------|
| 과실비율 | 히어로 카드 (그라데이션 배경, 큰 숫자, 비율 바) |
| 상세 분석 | 캐러셀 (섹션별 카드, 스와이프/탭 전환, 고정 높이+내부 스크롤) |
| 사고 장면 | 영상: 실제 프레임 + 2D 재현도 나란히 / 텍스트: 2D 재현도 |
| 감지 통계 | 4칸 그리드 (차량, 보행자, 충돌, 차선 수) |
| 액션 | 결과 복사, 이미지 저장 |

## 4. 충돌 감지 알고리즘

### 4.1 현재 구현
```
충돌 판정 = (IoU > 0.2) 
         OR (IoU > 0.05 AND 거리 < 60px AND 차량 속도 < 5px/s)
```

### 4.2 사고 순간 탐지
```
1단계: 연속 2프레임 이상 충돌 감지 → 첫 프레임 = 사고 순간 (확정)
2단계: 연속 충돌 없으면 점수 기반 fallback
       - 충돌 시작 순간: +50점
       - 전복 발생: +30점
       - 차량 수 급변: +8점/대
       - 기본 차량 수: +1점/대
```

### 4.3 향후 개선 계획
- Optical Flow 기반 급격한 움직임 감지
- 사고 분류 모델 (DoTA/CCD 데이터셋 fine-tuning)
- 사고 전후 구간 분석 (정상→사고→정지 패턴)

## 10. 리스크 및 제약사항

| 리스크 | 영향 | 완화 방안 |
|--------|------|----------|
| AI 과실비율 오판 | 사용자 법적 불이익 | "참고용" 면책 고지 강화 |

| Flask 서버 별도 운영 필요 | 인프라 복잡성 | Railway/Render 등 간편 배포 활용 |
