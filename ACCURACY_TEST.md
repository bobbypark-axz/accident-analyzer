## 분석

### 강점
- **명확한 과실 사례 정확도 높음**: 신호위반(100:0), 추돌사고(0:100), 주차장 후진(70:30), 유턴(80:20) 등 판례가 명확한 유형은 정확히 맞춤
- **전체 90%가 10pp 이내**: 대부분의 사례에서 실용적으로 활용 가능한 수준의 정확도

### 개선 필요 사항
- **과실 당사자에게 더 가혹한 경향**: 6건 중 5건에서 주요 과실자에게 기준보다 10pp 더 높은 과실 부여 (보수적 편향)
- **최악 케이스 (테스트 #6)**: 직진(녹색) vs 같은 방향 우회전 — AI가 직진 차량에 20% 과실을 부여했으나 기준은 0%. 같은 방향 진행 사고에서 직진 우선권을 과소평가
- **보행자 사고 (테스트 #10)**: 녹색 신호 보행자에게 10% 과실 부여 — 기준상 0%여야 함

### 편향 패턴
```
AI 경향: 주요 과실자에게 기준보다 ~10pp 더 가혹하게 판단
원인 추정: 프롬프트가 "양쪽 모두의 과실"을 찾으려는 경향
```

## 개선 방안

1. **시스템 프롬프트 보강**: 명확한 과실 기준 (0:100, 100:0)인 사례에서는 상대방 과실을 0으로 유지하도록 지시
2. **도표번호 매칭 강화**: 손보협 도표번호별 기본 과실비율을 프롬프트에 포함
3. **보행자 사고 규칙 추가**: 녹색 신호 보행자는 원칙적으로 과실 0% 명시
4. **RAG 연동**: Supabase 벡터 검색으로 실제 판례 참조하여 정확도 향상
